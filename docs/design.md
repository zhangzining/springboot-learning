# 功能性需求

### 1. 用户管理

1.1 用户
- 用户分为普通用户和企业用户
- 用户可以通过如下方式注册和登录，密码要求8-16位必须包含：数字、大写字母、小写字母、特殊字符(@$%!~)，考虑密码传输安全
  - 手机号+验证码(可以使用单独的模拟接口来模拟发送短信)
  - 手机号+密码
  - 邮箱+密码
- (可选)用户可以通过短信或者邮箱的方式重置密码，且每次重置间隔时间大于15天

1.2 普通用户
- 普通用户可以设置 头像、昵称、生日、所属地、性别等属性

1.3 企业用户
- 企业用户可以设置 头像、昵称、生日、所属地、性别等属性，在展示时会附带企业信息
- 企业用户需要有一个企业管理员，需要通过企业管理员才可以管理企业信息
- 企业管理员需要提供 公司名、注册地、统一社会信用代码、企业营业执照照片
- 企业信息需要系统管理员认证，认证通过前，企业下所属账号都无法发布任何信息
- 企业管理员可以管理企业用户
  - 通过 手机号/邮箱 邀请普通用户加入企业成为企业用户
  - 企业管理员可以手动删除企业用户
  - 企业用户可以提交退出申请，在企业管理员审核通过后退出企业变回普通用户

1.4 系统管理员
- 系统管理员可以通过邮箱+密码的方式登录，需要有且只有一个超级系统管理员
- 超级系统管理员可以管理系统管理员
  - 通过邮箱创建系统管理员，给予默认密码，默认状态为禁用
  - 修改系统管理员账号 启用/禁用 状态
  - 删除系统管理员
  - 修改系统管理员权限
- 系统管理员可以管理用户(考虑不同角色)
  - 审核企业信息通过后，企业所属企业用户可以进行信息发布等所有操作
  - 封禁/解封企业用户
  - 封禁/解封普通用户
